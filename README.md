# Demo Login Page

This is a simple static login page demo (HTML / CSS / JavaScript).

Features:
- Accessible labels and form structure
- Client-side validation (email format + minimum password length)
- Password show/hide toggle
- Simulated authentication (demo credentials: `user@example.com` / `Password123`)

How to run:
1. Open `index.html` in your browser (no server needed).
2. Use the demo credentials or enter your own — you'll see success/failure messages.

Notes:
- Replace `fakeAuth` in `script.js` with a real `fetch` call to your backend for production.
- This demo is client-side only and not secure for real authentication (no hashing, no HTTPS requirements, etc.).

## Oracle backend (store credentials in Oracle DB)

This repository includes a minimal Express backend (`server.js`) that accepts credentials from the frontend and stores them in an Oracle database. The server hashes passwords (using bcrypt) before saving.

Quick setup:

1. Install dependencies:

   npm install

2. Create a `.env` file from `.env.example` and set the Oracle connection values:

   ORACLE_USER, ORACLE_PASSWORD, ORACLE_CONNECT_STRING (e.g. `dbhost:1521/ORCLPDB1`)

3. Create the table in your Oracle database (one-time):

```sql
CREATE TABLE credentials (
  id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name VARCHAR2(255),
  email VARCHAR2(255) UNIQUE,
  password_hash VARCHAR2(255),
  mode VARCHAR2(20),
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP
);
```

4. Start the server:

   npm start

5. Update `index.html`/`script.js` location or host so the browser can reach `http://localhost:3000` (CORS enabled by default). The frontend already posts to `/api/register` and `/api/login`.

Security notes:
- Passwords are hashed server-side with bcrypt.
- Use HTTPS and proper secret management in production.
- Add rate limiting, logging, and monitoring for real deployments.
